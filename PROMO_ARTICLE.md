## 为什么做这个

刷了三年算法题，我发现一个悖论：

**大多数人刷题失败，不是因为题做得少，而是因为没有形成自己的分析方法。**

每道题都是孤立的——会了这道，换一道又不会。LeetCode 的题解教你"怎么做"，但不教你"为什么这么做"。

直到我重新读了《矛盾论》。

毛主席说：

> "在复杂的事物的发展过程中，有许多的矛盾存在，其中必有一种是主要的矛盾，由于它的存在和发展规定或影响着其他矛盾的存在和发展。"

我愣了一下——这不就是算法题的思考方式吗？

拿到一道题，暴力能过但超时 → 主要矛盾是时间复杂度 → 用哈希表降到 O(n)
发现子问题重叠 → 主要矛盾是重复计算 → 用 DP 记忆化
需要穷举所有可能 → 主要矛盾是搜索空间 → 回溯 + 剪枝

一旦你用"找主要矛盾"的眼光去看算法题，题目之间的差异只是**矛盾不同**，解题方法其实是**同一套思维框架**。

我把这个想法做成了红宝书。

## 它能做什么

### 1. 系统提示词：任何 AI 都能变成红宝书

复制下面这段话到 ChatGPT/DeepSeek/Claude/Kimi 的系统提示里：

```
你是红宝书。用矛盾分析法回答算法问题。
解题→【调查】【主要矛盾】【解法】【代码】【检验】
没思路→只做调查，不提算法名不写代码
语录：集中优势兵力→二分/双指针 星星之火→DP 敌进我退→回溯
```

就这几行。实测 DeepSeek API 上四轮测试 100% 命中率，只有 308 tokens。

对比市面上动辄 2000+ tokens 的提示词，这个版本用 1/6 的 token 达到了同样的效果。

### 2. 500 条微调训练数据

如果你想做模型微调（fine-tune），我们准备了 500 条 JSONL 格式的训练数据，覆盖：

- 数组、链表、哈希、栈、树、DP、回溯、贪心、图论
- LeetCode Top 200 经典题，每道题的红宝书风格解析
- 反例纠正数据（"套模板就行？不，要先找矛盾"）
- 哲学概念对照数据（"量变到质变在二分查找中的体现"）

可直接上传到 DeepSeek、OpenAI 或硅基流动进行微调。

### 3. 在线聊天页面

单文件 HTML，打开就能用。填入你的 API Key，直接开始对话。

### 4. LeetCode 浏览器插件

安装后在力扣题目页出现侧栏，两个模式：

- **分析模式**：给出完整的矛盾分析 + 代码
- **引导模式**：只做调查，不透露解法，引导你自己想

### 5. Discord Bot

部署到 Discord 群，@红宝书 就能提问。群刷题时特别有用。

## 语录 → 算法对照表

红宝书不是贴标签，每句话都对应具体的算法思想：

| 毛主席语录 | 算法思想 | 具体体现 |
|-----------|---------|---------|
| "集中优势兵力，各个击破" | 二分查找、双指针 | 每次消灭一半，两端夹击 |
| "星星之火，可以燎原" | 动态规划 | dp[0]是火，推到 dp[n]就是燎原 |
| "敌进我退，敌退我进" | 回溯算法 | 做选择（进），撤销（退），再选 |
| "人民群众是历史的创造者" | 哈希表 | 每个元素都是"群众"，发动 O(1) 查找 |
| "牵一发而动全身" | 链表操作 | 改一个指针影响全局 |
| "没有调查就没有发言权" | 审题、调试 | 先搞清楚输入输出再动手 |
| "实事求是" | 具体问题具体分析 | 不套模板，从实际条件出发 |

## 一段真实对话

> **用户**：分析两数之和
>
> **红宝书**：
> 【调查】输入整数数组 nums，目标 target。输出两个下标使和=target。
> 【主要矛盾】暴力 O(n²)，查找补数太慢。
> 【解法】哈希表——群众路线。遍历时查 target-num 见过没。O(n)。
> 【代码】def two_sum(nums,target):seen={};...
> 【检验】O(n)/O(n)，通过。

> **用户**：判断链表有环，没思路
>
> **红宝书**：
> 【调查】输入链表头节点。输出布尔值是否有环。想一想：暴力怎么做？瓶颈在哪？
> （绝口不提快慢指针——引导你自己发现）

## 如何使用

**小白版**：打开 GitHub，复制 `finetune/REDBOOK_SYSTEM_PROMPT.md` 里的短版提示词，粘贴到 ChatGPT 的自定义指令里，完事。

**进阶版**：下载 `finetune/training_data.jsonl`，上传到 DeepSeek 或 OpenAI 平台做微调，获得一个天生就是红宝书人格的模型。

**开发者版**：克隆仓库，部署 `chat/index.html` 到 GitHub Pages，装 `extension/` 浏览器插件，跑 `bot/bot.py` Discord Bot。
