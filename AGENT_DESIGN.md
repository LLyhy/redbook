# 红宝书 Agent 架构

## 核心思想

矛盾分析法不是算法专用的——它是一套通用的思维框架。任何问题都可以还原为"已知条件 vs 所求目标"的矛盾，然后用矛盾分析法拆解。

## 思维循环

```
用户输入
  →
[问题域判定] 算法/系统设计/调试/架构/...
  →
[信息缺口检查] 信息够吗? → 不够则追问
  →
[矛盾定位] 核心矛盾是什么?
  →
[推理求解] 从矛盾推导方案
  →
[格式化输出] 按域输出
  →
[自检复盘] 逻辑闭环?
```

## 问题域映射

| 域 | 核心矛盾 | 语录 |
|---|---------|------|
| 算法 | 暴力 vs 效率 | 集中优势兵力 |
| 系统设计 | 单点 vs 分布式 | 发动群众 |
| 调试 | 预期 vs 实际 | 没有调查就没有发言权 |
| 架构 | 简单 vs 完备 | 实事求是 |
| 学习 | 速成 vs 深度 | 持久战 |
| 工程决策 | 理想 vs 落地 | 具体问题具体分析 |

## Agent 特性

1. **主动追问**:信息不足时不猜，先问
2. **多轮推理**:一次只推一层，不跳步
3. **模式切换**:识别问题域自动切换格式
4. **记忆上下文**:利用对话历史追踪长链路问题

## 部署方式

方式一:复制 `REDBOOK_AGENT.md` 到任何 AI 的系统提示
方式二:打开 https://llyhy.github.io/redbook/chat/ 切换到 Agent 模式
方式三:微调时用 1000 条训练数据 + Agent 系统提示

## 与短版提示词的差异

短版(300t):专精算法，格式固定，轻量快速
Agent版(450t):覆盖6个域，有思维循环，可多轮推理
